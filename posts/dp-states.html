前言

动态规划的核心在于状态定义。很多题目解不出来的原因不是想不到转移方程，而是状态定义本身就错了。本文总结常见的DP状态设计模式。

一、线性DP

最基础的DP形式，状态通常是 dp[i] 表示前i个元素的最优解。

1.1 最长上升子序列 (LIS)

状态： dp[i] 表示以 nums[i] 结尾的最长上升子序列长度

转移： dp[i] = max(dp[j] + 1)，其中 j < i 且 nums[j] < nums[i]

for i in range(n):
    for j in range(i):
        if nums[j] < nums[i]:
            dp[i] = max(dp[i], dp[j] + 1)

1.2 背包问题

状态： dp[i][w] 表示前i个物品，容量为w时的最大价值

转移：

dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])

二、区间DP

状态涉及区间的两个端点：dp[l][r] 表示区间 [l, r] 的最优解。

2.1 石子合并

状态： dp[l][r] 表示合并区间 [l, r] 的最小代价

转移： 枚举分割点 k

for len in range(2, n+1):
    for l in range(n-len+1):
        r = l + len - 1
        for k in range(l, r):
            dp[l][r] = min(dp[l][r], dp[l][k] + dp[k+1][r] + sum[l..r])

三、状态压缩DP

当状态可以用二进制表示时，用整数作为状态。

3.1 TSP 旅行商问题

状态： dp[mask][i] 表示访问集合为mask、当前在i的最小路径

转移：

for mask in range(1 << n):
    for i in range(n):
        if mask & (1 << i):
            for j in range(n):
                if not (mask & (1 << j)):
                    dp[mask | (1 << j)][j] = min(
                        dp[mask | (1 << j)][j],
                        dp[mask][i] + dist[i][j]
                    )

四、状态机DP

明确当前处于哪个"状态"，状态之间有转移规则。

4.1 股票买卖

状态： dp[i][0/1] 表示第i天不持有/持有股票的最大收益

dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])  # 不持有
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])  # 持有

五、树形DP

在树上进行DP，通常以节点为状态，子节点信息合并到父节点。

5.1 树的最大独立集

状态： dp[u][0/1] 表示以u为根的子树，u不选/选时的最大独立集

def dfs(u):
    dp[u][0] = 0
    dp[u][1] = 1
    for v in children[u]:
        dfs(v)
        dp[u][0] += max(dp[v][0], dp[v][1])
        dp[u][1] += dp[v][0]

总结

类型状态形式典型问题
线性DPdp[i]LIS、背包
区间DPdp[l][r]石子合并、回文
状态压缩dp[mask][i]TSP、棋盘覆盖
状态机dp[i][state]股票、自动机
树形DPdp[u][state]独立集、树直径
状态设计没有标准答案，但好的状态设计能让转移方程清晰简单。多做题积累经验最重要。
